1. Blocker: Crashed trainer recorded as status="ok" when train.log lacks
   an explicit NaN/OOM marker.

Status: FIXED.

Action: scripts/run_next_experiment.py:706-718 (failure branch in
_do_run). Previously `record["status"] = m.get("status", "failed")` let
parse_metrics's default "ok" downgrade an authoritative launcher
"failed" whenever the trainer SIGKILL'd / segfaulted without writing
"loss is nan" or "CUDA out of memory" to the log. New logic: the
launcher's "failed"/"timeout" status is the floor; parse_metrics may
only REFINE the cause upward in specificity ("nan" or "oom"). All other
parsed values cannot overwrite the failure label.

We did not change parse_metrics.py's "ok" default. Rationale: parse_metrics
is also called from the success path (rc == 0), where defaulting "ok" is
correct. The launcher is the right place to encode the rc==0/rc!=0
distinction.

2. Blocker: Lockfile cannot self-heal across launcher SIGKILL / pod
   preemption.

Status: FIXED in two places.

Action 1: scripts/run_next_experiment.py:367-396 acquire_lock(). On
FileExistsError, read the PID stamp, probe with os.kill(pid, 0). If
ProcessLookupError -> unlink and retry the O_EXCL acquire (logged so
the recovery is visible in train.log / agent_tick.log). If the PID is
alive, or unreadable, return None (conservative).

Action 2: scripts/agent_tick.sh:132-148 outer pre-check. Same probe:
kill -0 against the lockfile's PID. If alive -> skip tick (existing
behaviour). If dead -> rm -f and proceed. This stops cron from
silently no-op'ing for 30+ minutes after a preempt.

Verify (local; the stale-PID test fork-exits a child to get a
guaranteed-dead PID):

    python3 - <<'PY'
    import os, tempfile
    from pathlib import Path
    L = Path(tempfile.mkdtemp()) / "test.lock"

    def acquire_lock(lockfile=L):
        try:
            fd = os.open(str(lockfile), os.O_CREAT | os.O_EXCL | os.O_WRONLY, 0o644)
            os.write(fd, f"{os.getpid()}\n".encode()); return fd
        except FileExistsError:
            try: stale = int(lockfile.read_text().strip())
            except (OSError, ValueError): return None
            try: os.kill(stale, 0)
            except ProcessLookupError:
                lockfile.unlink()
                fd = os.open(str(lockfile), os.O_CREAT | os.O_EXCL | os.O_WRONLY, 0o644)
                os.write(fd, f"{os.getpid()}\n".encode()); return fd
            except PermissionError: return None
            return None

    # live-PID case
    L.write_text(f"{os.getpid()}\n")
    assert acquire_lock() is None
    L.unlink()
    # stale-PID case
    pid = os.fork()
    if pid == 0: os._exit(0)
    os.waitpid(pid, 0)
    L.write_text(f"{pid}\n")
    fd = acquire_lock()
    assert fd is not None
    os.close(fd); L.unlink()
    print("F2 stale-PID self-heal: OK")
    PY

5. High: Convergence halt is silently disabled if any HN-FPR value in
   the window is None.

Status: FIXED.

Action: scripts/run_next_experiment.py:345-365 (convergence branch in
check_halt_conditions()). Old logic took xattn_valid_all[-window:],
read hn_fpr_worst_stripped from each, and ONLY computed the trend if
`all(v is not None for v in worst_values)`. A single legacy or
partial row in the window suppressed convergence indefinitely.

New logic: flatten the entire xattn_valid_all to a series of
hn_fpr_worst_stripped values with Nones filtered out, then take the
last `window` of those VALID values to compute first vs best. If
fewer than `window` valid values exist overall, the halt is
correctly deferred. Legacy rows no longer hide a converged sweep.

Verify (synthetic histories, no torch):

    python3 - <<'PY'
    def check(hist, window=4, min_runs=6, thresh=0.005):
        if len(hist) < min_runs: return None
        s = [h.get("hn_fpr_worst_stripped") for h in hist]
        s = [v for v in s if v is not None]
        if len(s) >= window:
            recent = s[-window:]
            if (recent[0] - min(recent)) < thresh:
                return f"halt first={recent[0]:.4f} best={min(recent):.4f}"
        return None

    # Legacy row in window must no longer suppress halt:
    hist = [
        {"hn_fpr_worst_stripped": 0.030},
        {"hn_fpr_worst_stripped": 0.029},
        {"hn_fpr_worst_stripped": 0.0285},
        {"hn_fpr_worst_stripped": 0.028},
        {"hn_fpr_worst_stripped": 0.0278},
        {},  # legacy row
        {"hn_fpr_worst_stripped": 0.0275},
    ]
    r = check(hist)
    assert r is not None and "halt" in r
    print("F5 None-filter regression: OK")
    PY

6. High: Trainer subprocess group is not isolated -- timeout-kill
   leaks DDP workers and dataloader children.

Status: FIXED.

Action: scripts/run_next_experiment.py:455-484 (launch_trainer).
`subprocess.Popen(..., start_new_session=True)` puts the trainer in
its own process group. On timeout: `os.killpg(proc.pid, SIGTERM)`,
then `os.killpg(proc.pid, SIGKILL)` after a 30-second grace window.
Both calls are wrapped in `ProcessLookupError` handlers so a race
where the child already exited does not raise. This kills the WHOLE
tree -- the accelerate parent, its DDP workers, and any worker-pool
dataloader children -- on a single signal, rather than the prior
behaviour where SIGTERM hit only the parent and the children orphaned
under PID 1.

Verify (no GPU -- structural only):

    python3 - <<'PY'
    src = open("scripts/run_next_experiment.py").read()
    assert "start_new_session=True" in src
    assert "os.killpg(proc.pid, signal.SIGTERM)" in src
    assert "os.killpg(proc.pid, signal.SIGKILL)" in src
    print("F6 process group + killpg: OK")
    PY
