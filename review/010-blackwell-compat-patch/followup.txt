2. High: Preflight still allows bitsandbytes 0.43 even though Blackwell
   now requires 0.45+.

Status: FIXED.

Action: aligned all three version contracts at 0.45.0:

  - `scripts/preflight_check.py:check_bitsandbytes` default raised from
    `min_version: str = "0.43.0"` -> `"0.45.0"`. Added a docstring
    explaining why and pointing at this review.
  - RUNBOOK §10 line that said "`paged_adamw_8bit` requires
    `bitsandbytes>=0.43`" rewritten to "requires `bitsandbytes>=0.45.0`
    (Blackwell paged-kernel floor -- review 010)" plus a one-line
    reminder that requirements.txt and preflight agree.
  - requirements.txt already had `>=0.45.0,<0.46` from commit 925b997;
    unchanged here.

The reviewer's alternative -- making preflight
`torch.cuda.get_device_capability(0)`-aware so Hopper keeps a lower
floor -- DEFERRED. Rationale: we have no concrete Hopper deployment
right now (H100/H200 SXM both Unavailable on Secure Cloud, the actual
pod is RTX PRO 6000 Blackwell), and adding architecture-conditional
version logic to preflight is reversible later. A single floor of
0.45.0 is also harmless on Hopper because 0.45.x still has Hopper
kernels. If a future deployment needs to pin Hopper to bnb < 0.45 for
some reason, that's the time to add the conditional.

   trainer factory.

Status: FIXED.

Action: implemented `adamw_8bit` in `build_optimizer`
(`src/train/common.py:227-258`) so the RUNBOOK §10 fallback recipe is
actually executable from a trainer config. The build_optimizer accept
list now reads: paged_adamw_8bit, adamw_8bit, adamw.

Code change:

Also tightened the unknown-optimizer error message to list the three
supported values, so a typo in a config produces a useful failure

Docstring rewritten to document all three modes with memory-footprint
guidance: paged_adamw_8bit (default, ~5 B/param), adamw_8bit (fallback,
~5 B/param non-paged), adamw (full fp32, ~12 B/param, last resort).

RUNBOOK §10 recovery recipe updated in lockstep:
  - Old text: "fall back to non-paged 8-bit optimizer (`AdamW8bit`
    instead of `PagedAdamW8bit`) in the trainer configs"
  - New text: "set `optimizer: adamw_8bit` in the trainer config

adamw_8bit path is the documented next step and is now executable.
