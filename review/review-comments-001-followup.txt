6. Medium: opaque eval mode is not actually neutral.

Status: FIXED.

The reviewer's argument was correct: the previous stable journey/actor ID
mapping meant the model could learn the mapping during training-time
eval-mode dropout (50% full / 25% opaque / 25% stripped), making opaque
effectively just renamed full.

Change in eval/eval_modes.py:

  - apply(text, mode, rng=None) now accepts an optional random.Random.
  - In opaque mode, the function generates a per-call random permutation
    of IDs [01..99] for journey families and another for actor families,
    then assigns IDs on-the-fly per family encountered in the text.
  - Within a single call, the same journey name maps to the same ID, so
    open and close tags pair correctly and the model retains the
    structural (boundary) signal — but across calls, the ID for a given
    name is randomized.

Change in src/train/mixers/eval_mode_dropout.py:

  - EvalModeDropoutCollator._transform now constructs a fresh
    random.Random per opaque example (seeded from the collator's master
    RNG) and passes it to eval_modes.apply().
  - For full and stripped modes, no rng is passed (no-op).
