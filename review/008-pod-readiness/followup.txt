2. Blocker: The documented default auto-loop script does not exist.

Status: FIXED.

Action: created `scripts/agent_tick.sh` with the reviewer's exact
spec:

  - Derives REPO_ROOT from script location (matches Finding #1's
    portability fix).
  - Activates `/workspace/.venv` (configurable via `AGENT_VENV` env).
  - Calls `python scripts/run_next_experiment.py --halt-check` first;

5. High: External backup will not copy the actual x-attn checkpoints.

Status: FIXED — both subbugs.

Sub-bug A (top_3 YAML parser): the embedded `python -c` printed each
top_3 entry as a Python dict (`{'exp_id': 'exp_001', ...}`); the
shell for-loop over whitespace-tokenized that into garbage. Fixed by
emitting JUST `entry["exp_id"]` in the heredoc parser. Verified
locally:

    python3 -c "
    fake_state = {'top_3': [{'exp_id': 'exp_001', 'auc_stripped': 0.8},
                            {'exp_id': 'exp_002', 'auc_stripped': 0.79}]}
    for entry in fake_state.get('top_3', []):
        if isinstance(entry, dict) and 'exp_id' in entry:
            print(entry['exp_id'])
    "
    # Output:
    #   exp_001
    #   exp_002

Sub-bug B (checkpoint location): backed up the wrong path. The
trainer (`train_xattn.py:282`) saves `xattn_state.pt` to the run
directory at `src/auto_research/runs/exp_NNN/`. There is no
`/workspace/checkpoints/xattn_NNN/` produced anywhere. Fixed by
syncing `$REPO_ROOT/src/auto_research/runs/$exp_id/` directly for
each top-3 run. That picks up `xattn_state.pt`, `metrics.json`,
`ci_report.json`, `gate_trajectory.json`, `leakage_report.json`, and
`config.yaml` — everything needed to resurrect a top-3 run on a new
pod.

6. Medium: Fresh clones do not contain the state files the agent
   instructions require reading.

Status: FIXED.

Action — three parts:

a. New script `scripts/init_auto_research_state.py`:
   - Creates `src/auto_research/experiments.jsonl` (empty file).
   - Writes empty-history `src/auto_research/sweep_state.yaml`
     (schema_version=1, n_completed=0, current_best=null, halted=false).
   - Idempotent: refreshes timestamp on existing empty state; refuses
     to overwrite non-empty state unless --force.
   - Creates `src/auto_research/runs/` dir if missing.

b. `RUNBOOK.md` §1.3 bootstrap now calls
   `python scripts/init_auto_research_state.py` between `pip install`
   and `preflight_check.py`. The agent loop can therefore expect both
   state files to exist on every fresh boot.

c. `src/auto_research/AGENT_INSTRUCTIONS.md` per-iteration loop step 1
   updated:
       If `sweep_state.yaml` or `experiments.jsonl` is missing
       (gitignored runtime state — fresh clone or just-reset), treat
       it as zero completed runs and proceed normally. The first
       launcher invocation will create both files. If uncertain,
       run `python scripts/init_auto_research_state.py`.
   Belt-and-braces: the agent now handles the missing-file case
   gracefully even if RUNBOOK §1.3 was skipped.
