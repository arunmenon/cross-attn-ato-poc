6. DISPUTES audit-014 finding #4 (torch.save atomicity over-severed).

Status: ACCEPTED, but the fix was bundled anyway.

Codex's reasoning is correct: backup_to_external.sh's top-3 ordering
means the in-flight run is never targeted by the backup during the
torch.save window, so the worst case is a torn LOCAL .pt that the
launcher and operator can rebuild from base + redo, not a torn
EXTERNAL sync that ships partial weights off-pod. The High framing
overstated the externalized risk.

We bundled the 3-line tmp+rename anyway (audit-014 followup F4) for
two reasons: (a) zero cost, matches the atomic-write pattern already
in use for metrics.json + sweep_state.yaml + experiments.jsonl; (b)
the "current run is never in top_3 mid-write" invariant is true today
but is one backup-script edit away from breaking. Bundling now means
no one re-debates this in a future review.
