3. Medium: CODEX_PROMPT.md's py_compile command fails on doc-only diffs.

Status: FIXED.

Action: review/CODEX_PROMPT.md Step 5 replaced the unsafe one-liner
with an empty-safe block:

    PY_FILES=$(git diff --name-only "${CLOSING_COMMIT}..HEAD" | grep -E '\.py$' || true)
    if [ -n "$PY_FILES" ]; then
        python3 -m py_compile $PY_FILES
    else
        echo "no Python files changed; py_compile skipped"
    fi

`|| true` after grep prevents the assignment from inheriting a non-zero
exit when no matches are found, which would otherwise be a problem
under `set -e`.

Verify (from repo root):

    if [ -n "$PY_FILES" ]; then python3 -m py_compile $PY_FILES; else echo "no Python files changed; py_compile skipped"; fi
(Should print "no Python files changed; py_compile skipped" against the
current head, which has only doc commits since b903809.)

Notes on the two-commit pattern (review 003 closure):

This followup is committed alongside the protocol fixes (commit A).
Then INDEX.md is updated to record commit A's hash in the 003 row in a
