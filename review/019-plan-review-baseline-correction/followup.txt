1. Blocker: The tie-aware metric formula rounds away the exact target.

Status: FIXED in plan.

Action: Plan Task 1 ("metric-score-risk") rewritten. The formula
description in the original plan used `need = round(target_fpr *
n_legit)`. Per Codex's verification run, with `n_legit = 2979`
(clean eval), this rounds 29.79 -> 30, yielding `event_only clean
worst = 0.073529` instead of the 0.0730 reference value. The plan's
own acceptance criterion AC1 requires reproduction within 1e-4, which
the rounded formula fails.

The revised Task 1 specifies:
  - `need = target_fpr * n_legit` (FLOAT — explicit "do not round").
  - `alpha = (need - n_above) / n_tied` when `n_tied > 0 and need >
    n_above`, else `alpha = 0.0`.
  - Worked example for event_only clean: `n_legit = 2979`, `need =
    29.79`, `T = -9.9375`, `n_above = 4`, `n_tied = 141`,
    `alpha = (29.79 - 4) / 141 = 0.18293...`, reproducing
    `event_only clean worst = 0.0730` within 1e-4.

Verify (post-implementation):
    python3 -m eval.score_risk --selftest
    # Plus the worked-example assertions in AC1.

4. Medium: The future-regeneration split plan can miss class-balance targets.

Status: FIXED in plan.

Action: Plan Task 6 rewritten. The original "hash[:2] mod N" approach
locked the split assignment before within-family stratification, so
class balance per `journey_family` could drift from the requested
`eval_frac`.

The revised gate uses **class-balanced disjoint stratification**:
  1. Compute `structured_events_hash` per row.
  2. Group rows by `(journey_family, structured_events_hash)`.
  3. For each `journey_family`, walk groups in deterministic order
     (sorted by group hash). Assign each WHOLE GROUP to eval until
     the family's eval-row-count / family-total reaches `eval_frac`;
     remaining groups go to train.
  4. Invariant preserved: a `structured_events_hash` never appears
     in both splits (groups are atomic).
  5. `build_summary.json` gains: `requested_eval_frac`,
     `actual_eval_frac_overall`, per-family
     `{actual_eval_frac, n_train, n_eval}`,
     `n_unique_structured_events_train`,
     `n_unique_structured_events_eval`,
     `n_overlap_structured_events: 0` (invariant check).

AC6 updated to reference the new approach and the build_summary
additions.

Verify (post-implementation, requires a regeneration which is
explicitly OUT of scope for this plan):
    # Future regeneration only:
    python3 data/gen/build_dataset.py --n 1000 --enforce-disjointness
    cat data/<output>/build_summary.json
    # Expected: n_overlap_structured_events: 0;
    # actual_eval_frac_overall ≈ requested_eval_frac within 1%
